

Dev Phases:
Recommended Build Order (Solo Dev, Ship Fast)
Six phases. Each phase is shippable on its own.

Phase 0 — Foundation (Week 1)
Goal: Replace the starter with the TechTivAI shell.

Design tokens in globals.css

Colors: #04060d, #080d1c, cyan/lime/violet/rose, muted text
Tailwind @theme mapping — no hardcoded hex in components
Typography

Clash Display (headings), Cabinet Grotesk (body), Instrument Serif (accent) via Fontsource
Remove Geist from layout.tsx
Folder structure (from cursor-context.md)

app/(pages)/...
components/{ui,sections,shared,animations,three}
lib/
prisma/
Base UI

Button, Card, Badge, Input, GlassPanel
Navbar, Footer, mobile menu
Layout & metadata

SEO defaults, Open Graph, dark theme only
SectionWrapper + Framer Motion stagger helper
.env.example with all env vars from cursor-context

Exit criteria: Branded shell, nav works, placeholder homepage sections stack vertically.

Phase 2 — Data Layer & Leads (Week 4)
Goal: Persist interactions before wiring expensive AI.

Prisma schema (minimum viable):

Lead — contact info, source, discovery answers (JSON)
AiSession — type (chat/discovery/voice), messages, metadata
Proposal — job_id, status, r2_url
User — admin (Better Auth)
Neon dev DB + lib/prisma.ts singleton

POST /api/leads — Zod validation, save lead

Wire discovery wizard + contact form → API

PostHog — page views, CTA clicks, wizard completion

Exit criteria: Completing discovery saves a lead; you can see it in Prisma Studio.

Phase 3 — AI Core (Week 5–6)
Goal: Real intelligence on discovery + hero chat.

Dependencies: ai (Vercel AI SDK), openai, @anthropic-ai/sdk, zod
lib/ai.ts — server-only clients
POST /api/ai/recommend — discovery answers → recommendations + ROI (GPT-4o)
POST /api/ai/chat — streaming chat for hero + discovery
POST /api/pricing/calculate — server-side pricing logic
Log every session to AiSession
Start with rule-based pricing fallback if AI is slow; refine prompts later.

Exit criteria: Discovery wizard returns real AI recommendations; hero chat streams; pricing updates from API.

Phase 4 — Voice, Proposals & Auth (Week 7–8)
Goal: High-value conversion features + admin access.

Better Auth — Google OAuth, app/api/auth/[...all]/route.ts
Vapi — POST /api/voice/session, embed voice UI in Hero + Contact
Proposals — POST /api/proposals/generate → n8n webhook → return job_id → poll status
Cloudflare R2 — store generated PDFs
/dashboard — protected route: leads list, session logs, proposal status
Exit criteria: Voice call works end-to-end; proposal job triggers n8n; admin can log in and view leads.

Phase 5 — Secondary Pages & Polish (Week 9–10)
Goal: Complete site map + SEO.

Page	Priority
/discovery
Full-screen onboarding (wizard v2)
/services
9 service detail sections
/industries/[slug]
8 dynamic pages (start with 3–4)
/pricing
Standalone pricing engine
/case-studies
Expandable stories
/contact
Form + voice + calendar embed
/resources
Blueprint only — blog MDX or defer to Phase 6
Plus: performance (lazy load Three.js/GSAP), accessibility pass, Lighthouse tuning.

Phase 6 — Launch & Growth (Week 11+)
Vercel production + Neon + Prisma Accelerate
Upstash rate limiting on AI routes
LangGraph pipeline for multi-step proposals (n8n)
CRM webhooks (HubSpot/GHL)
Content/SEO (resource center, industry landing pages)
Homepage Section Order Note
Blueprint and cursor-context.md differ slightly:

Blueprint: Hero → Trust early (section 2)
cursor-context: Trust at section 8 (after value props)
Recommendation: Follow cursor-context for build order (value before trust), but you can move Trust to position 2 later if you want faster credibility on first scroll.
